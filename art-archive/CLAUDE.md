# The Art Archive

Status: In Development
Audience: Researchers, collectors, and data scientists studying auction market dynamics
Brand tone: Academic archive meets fine-art editorial — precise, dispassionate, data-forward

## Key sections
- Catalogue (`index.html`) — filterable grid of sold auction lots (artist, hammer price, estimate, performance)
- Analysis (`analysis.html`) — Chart.js visualisations: price tiers, top artists, house performance

## Data files
- `catalogue.json` — all lot metadata (generated by `scripts/build_art_json.py`)
- `analysis.json` — pre-computed statistics for charts

## Database
- `data/art.db` — SQLite, `art_items` table — fully isolated from `data/hermes.db`
- Schema: `scraper/art_db.py`

## Scrapers
- `scraper/christies.py` — Christie's sold lots via public search API
  - Run: `uv run python -m scraper.christies --max-lots 300` (from repo root)
  - Keywords: painting, sculpture, works on paper, photography, print

## Build pipeline
```bash
uv run python -m scraper.christies --max-lots 300
uv run python -m scripts.build_art_json
```

## Filter axes
- Sale performance: above / within / below estimate
- Medium category: painting, works on paper, photography, print, sculpture, other
- Price tier: <$10k / $10k–100k / $100k–1M / $1M+
- Auction house (dynamic from data)
- Free-text artist/title search

## Notes
- All prices normalised to USD using static FX rates in `scraper/christies.py` (`_FX` dict)
- `sale_performance` is computed at insert time in `art_db.py::_sale_performance()`
- Image URLs are taken directly from source CDN — no local copies
- `analysis.html` is Phase III work; the nav link exists but the page is not yet built
